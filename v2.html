Key Findings

27B: MTP is a massive win. n=2 gives ~95% speedup over baseline (7.8 → 15.2 t/s). Poll/ub flags have NO effect — pure MTP is optimal.
35B: --poll and -ub actively hurt performance. On b9484, the 35B model with MTP + --poll/-ub drops to ~7-8 t/s (50% slower). The ONLY config that works is bare --spec-type draft-mtp --spec-draft-n-max 2 --spec-draft-ngl 99 -t 16 without poll or ub. This is a regression vs b9198 where these flags helped.
n=5 hurts both models. n=3 is marginally better on 27B but worse on 35B. Stick with n=2.

b9484 uses shared library architecture (libllama-cli-impl.so) vs monolithic b9198 binary. This may explain behavioral differences with --poll/-ub flags.
